# PM Scenario Coach Skill

一个面向产品经理实习、校招和新人面试的场景题轻量陪练 skill。

它是一个用对话带你练判断标准的面试教练：根据目标公司、部门和产品方向生成或拆解场景题，并按照 STAR 法则一步步训练。

## 适合谁

- 正在找产品实习、校招产品、产品管培岗位的同学
- 场景题总是答得很泛的人
- 一上来就想方案，容易说成“发券、改入口、做活动”的人
- 想练腾讯、字节、小红书、美团、京东等公司风格题的人
- 想练用户产品、数据产品、AI 产品、增长产品、商业产品方向的人

## 它怎么练

Skill 会按照 STAR 法则拆题：

```text
S：场景，判断这道题发生在什么真实业务情境里
T：目的，判断这道题真正要解决什么问题
A：行为，选择更符合公司调性的产品动作
R：结果，判断用什么指标证明方案有效
```

每一步都不是让你写长文，而是先给选项，让你选出认为合适的所有选项。

它会即时反馈：

- 你选对了什么
- 哪个选项容易带偏
- 这家公司真正看重什么
- 这一步可以迁移到什么题

最后会生成一段面试时可以直接开口的逐字稿。

## 支持的公司画像

当前内置：

| 公司/业务 | 重点判断标准 |
|---|---|
| 腾讯 / QQ | 用户价值、关系链、低压力表达、长期社交体验 |
| 字节 | 数据敏感、推荐分发、快速实验、增长效率 |
| 小红书 | 用户心理、社区真实感、内容质量、种草链路 |
| 美团 | 本地供需、履约效率、成本约束、多角色平衡 |
| 京东 | 零售信任、供应链履约、服务确定性、交易转化 |

## 支持的产品方向

- 用户产品
- 数据产品
- AI 产品
- 增长产品
- 商业产品

Skill 会把“公司调性 × 产品方向”组合起来出题。

比如：

```text
腾讯 QQ / 用户产品
```

会更偏社交关系、低压力表达、兴趣群融入。

```text
字节 / 数据产品
```

会更偏指标体系、实验分析、异常诊断。

```text
小红书 / AI 产品
```

会更偏内容创作、用户信任、AI 工具 adoption 和社区真实感。

## 安装

把 `pm-scenario-coach` 文件夹复制到 Codex skills 目录：

```bash
cp -R pm-scenario-coach ~/.codex/skills/pm-scenario-coach
```

然后重新打开或刷新 Codex，让新 skill 被加载。

## 使用方式

### 让它出题

```text
$pm-scenario-coach 我想练腾讯 QQ 部门用户产品场景题
```

```text
$pm-scenario-coach 我想练小红书 AI 产品方向
```

### 带着题来练

```text
$pm-scenario-coach 帮我练这道题：如何提升 QQ 群新用户首次发言率？
```

```text
$pm-scenario-coach 帮我拆一下：如何提升小红书普通用户发布笔记的意愿？
```

如果你带着具体题目来，skill 不会重新出题，会直接进入 STAR 拆解。

## 示例流程

用户：

```text
$pm-scenario-coach 我想练腾讯 QQ 部门产品经理
```

Skill：

```text
好，我们来练一题：腾讯 QQ / 用户产品。

场景题：
假设你是 QQ 产品经理，发现很多新用户加入 QQ 频道或兴趣群后，只浏览不发言，24 小时内首次互动率偏低。你会如何提升新用户的首次有效互动率？

我们按照 STAR 法则来分析，也就是：场景 × 目的 × 行为 × 结果。

腾讯 QQ 更看重用户价值、关系链、低压力表达和长期社交体验。请你带着这个认知开始下面的思考。

第一步：S，场景识别。
从下面选项中，选出你认为合适的所有选项：
...

直接回复你认为合适的所有选项。
```

完整示例见：[examples/tencent-qq-practice.md](examples/tencent-qq-practice.md)

## 设计原则

### 1. 轻量

不要求用户一上来写完整答案。先用选择题降低练习门槛。

### 2. 不让用户靠猜

正确选项不会固定在前面。每一步选项都要求打乱顺序。

### 3. 让用户学会判断标准

每一步反馈都必须给一条可迁移的判断方法。

### 4. 公司调性优先

同样是“提升转化”，腾讯、小红书、美团、字节、京东的好答案不一样。

### 5. 有情绪价值，但不过度卖萌

正确、差一点、完全错误都有轻量文字表情：

```text
正确：:D / 稳了 :D / 抓到重点了 :)
差一点：差一点 :) / 方向有了 :| / 这里有个小坑 :|
完全错误：先别急 :/ / 被带偏了 :/ / 我们拆开看 :/
```
